  <action>
  **Brand Icon:**
  Create an abstract geometric brand icon as an SVG. Design principles:
  - Simple geometric shape: an abstract "S" letter form or bracket-like symbol suggesting "sidekick" assistance
  - Single path SVG, minimal complexity (works at 16x16)
  - Use the blue accent color: `oklch(0.6 0.18 260)` or its hex equivalent (~#4F6EF7 or similar vivid blue)
  - Clean, modern, geometric — think Linear's corner bracket, Vercel's triangle, or Raycast's sparkle
  - Must work as: favicon (16x16, 32x32), sidebar inline mark, and potentially app icon

  Create `sidekiq-webapp/public/favicon.svg` with the icon.
  Also create `sidekiq-webapp/public/favicon.ico` from the SVG if feasible, or update the metadata to reference the SVG.

  Update `sidekiq-webapp/src/app/layout.tsx` metadata to reference the new favicon:
  ```tsx
  icons: [{ rel: "icon", url: "/favicon.svg", type: "image/svg+xml" }],
  ```

  **Codebase Sweep:**
  Run a comprehensive grep for remaining hardcoded colors in src/:
  ```bash
  grep -rn "zinc-\|bg-white\b\|text-white\b\|glass\|backdrop-blur" sidekiq-webapp/src/ --include="*.tsx" --include="*.ts" | grep -v node_modules | grep -v ".next"
  ```

  Fix any remaining instances found. Replace:
  - `bg-white` -> `bg-background` or `bg-card`
  - `text-white` -> `text-foreground` or `text-primary-foreground` (context-dependent)
  - Any remaining `zinc-*` -> appropriate semantic token
  - Any remaining `glass` or `backdrop-blur` -> remove

  Note: Some `bg-white` and `text-white` usage may be intentional (e.g., on colored backgrounds where white text is needed). In those cases, use `text-primary-foreground` (white on primary) or keep as-is if it's within a specific colored context (like an avatar with dynamic background color).
  </action>
