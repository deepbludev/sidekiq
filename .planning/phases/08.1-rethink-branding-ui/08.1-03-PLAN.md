  <files>

    sidekiq-webapp/src/components/auth/reset-password-form.tsx
  </files>
  <action>
  Read each auth component file and replace ALL hardcoded zinc-* color classes with semantic tokens. Apply this replacement map consistently:

  **Color replacements:**
  - `bg-zinc-900/50` or `bg-zinc-900/*` -> `bg-card`
  - `bg-zinc-800/50` or `bg-zinc-800/*` -> `bg-muted`
  - `bg-zinc-950` -> `bg-background`
  - `bg-zinc-900` -> `bg-card`
  - `border-zinc-800` or `border-zinc-700/50` or `border-zinc-700/*` -> `border-border`
  - `border-zinc-200/50` or `border-zinc-200/*` -> `border-border`
  - `text-zinc-400` or `text-zinc-500` -> `text-muted-foreground`
  - `text-zinc-200` or `text-zinc-100` -> `text-foreground`
  - `text-zinc-300` -> `text-muted-foreground`
  - `text-white` -> `text-foreground` (or `text-primary-foreground` if on colored bg)
  - `from-zinc-900 to-zinc-950` (gradients) -> replace with solid `bg-background`
  - `hover:bg-zinc-800` -> `hover:bg-accent`
  - `hover:text-zinc-300` -> `hover:text-foreground`
  - `placeholder:text-zinc-500` -> `placeholder:text-muted-foreground`

  **Additional auth-specific changes:**
  - Remove any `glass`, `glass-subtle`, `glass-input` class references
  - Replace `backdrop-blur` with nothing (solid backgrounds)
  - If auth-card has brand name display (e.g., "Sidekiq"), change to lowercase "sidekiq"
  - OAuth buttons should use the outline button style: `border border-border bg-transparent hover:bg-accent`
  </action>
  <verify>
  Run: `grep -r "zinc-" sidekiq-webapp/src/components/auth/` — should return no matches.
  Run: `grep -r "glass\|backdrop-blur" sidekiq-webapp/src/components/auth/` — should return no matches.
  Run: `cd sidekiq-webapp && pnpm tsc --noEmit 2>&1 | head -20` — no TypeScript errors.
  </verify>
  <done>
  All 6 auth component files use semantic tokens exclusively. No hardcoded zinc colors, no glass classes, no backdrop-blur. Brand name shows as lowercase "sidekiq".
  </done>
